# Design Pattern

Berisi `tips` bagaimana cara menulis code secara rapi agar maintainable dan debuggable.

## Komponen

Dalam membuat sebuah halaman dengan banyak widget, lebih baik dipisahkan dalam komponen. Dalam hal ini, komponen adalah sebuah class seperti `StatelessWidget` atau `StatefullWidget`, lebih baik daripada membuat builder function yang mengembalikan `Widget`. Dikarenakan:

- class memiliki `identity` pada widget tree, sehingga debugging lebih mudah.
- function Tidak bisa const
- class memiliki `Context` & Inheritance (Theme, MediaQuery, Provider, dll.)
